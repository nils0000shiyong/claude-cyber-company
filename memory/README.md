# Shared Memory Protocol

这个目录是Claude和Codex唯一允许共享的长期记忆层。

## 目录

- `stable/`: 已验证、可长期复用的信息。写入要谨慎。
- `working/`: 当前任务的阶段性上下文、未验证假设、草稿。
- `inbox/`: 两个平台之间的交接记录。只写必要摘要,不要写完整聊天历史。

## 读取规则

1. 复杂任务开始前先读本文件。
2. 只按任务需要读取相关记忆,不要全量读取整个`memory/`目录。
3. 不要把`.claude/`或`.codex/`当作共享记忆来源。

## 写入规则

1. 事实进入`stable/`前必须可验证,并标注日期、来源、置信度。
2. 未确认内容只能写进`working/`。
3. 平台交接只写进`inbox/`,格式为:日期、来源平台、任务、结论、待办。
4. 发现冲突记忆时不要覆盖,新增一条冲突说明并交给董事长裁决。
